Cache storm

100 identical cache-eligible requests, 20 at a time. The first wave misses and pays ~40ms of (simulated) inference; the rest are served from Redis in ~1ms.

idle

Idempotency replay

Eight requests race in with the same fresh Idempotency-Key — one executes, the rest get 409. Then twelve replays of the completed key return the stored response with zero new backend calls.

idle

Batch flood

Submits one async job with 100 items. The gateway writes job + items + outbox in one transaction and publishes to a Redis Stream; a separate worker process drains it on a bounded pool. Polls the job to a terminal status.

idle

Live traffic running

A steady ~4 req/s of uncacheable traffic. Turn it on, then kill the primary backend in a terminal — docker compose stop mock-llm-fast — and watch the breaker tiles flip while requests keep succeeding via failover. (docker compose start mock-llm-fast to recover.)

idle

Backend outage (CLI)

The full scripted version — stops the container, drives 150 requests, shows the breaker open, then restarts it and shows recovery. A browser can't stop containers, so this one runs in a terminal:

bash scripts/demo.sh backend-outage

Rate-limit burst (CLI)

Recreates the gateway with a 10 QPS per-key window, fires 50 rapid requests, counts the 429s, restores the default. Also terminal-only (it recreates a compose service):

bash scripts/demo.sh rate-limit-burst
